조합원에게 발송한 총회 안내, 리마인드, 결과 안내 등의 **전송 기록과 전송 결과를 확인하는 화면**입니다.  
전송이 실패한 경우 실패 사유를 확인하고 **즉시 재전송**할 수 있습니다.

---
## 설명  
  ![](assets/img/asemblyhistory.png)
  
조합원총회 상세 화면의 **전송이력 탭**에서는 조합원에게 발송된 이메일 및 알림톡(SMS)의 전송 기록과 성공 여부를 확인할 수 있습니다.  
각 전송은 **수신자 단위로 이력이 생성되며**, 실패한 경우 해당 행에서 바로 재전송할 수 있습니다.  
  
---  
  
### 1. 전송 이력 생성 시점

전송 이력 행(row)은 사용자가 **총회 안내, 리마인드, 결과 안내 등의 전송 버튼을 클릭한 시점**에 먼저 생성됩니다.

이후 시스템에서 총회 정보 변경 사항을 서버에 반영한 뒤 외부 전송 서비스(API)를 호출하여 실제 이메일 또는 알림톡을 발송합니다.

따라서 다음과 같은 순서로 처리됩니다.

1. 전송 버튼 클릭
2. 전송 이력 행 생성
3. 서버 데이터 반영
4. 이메일 / 알림톡 실제 발송
5. 발송 완료 시 전송일시 기록

전송 이력이 먼저 생성되기 때문에 **실제 발송이 완료되기 전까지 전송일시는 빈칸으로 표시될 수 있습니다.**

> 실제 전송 시간이 반영된 전송일시는 **화면을 새로고침하면 확인할 수 있습니다.**
> 이메일과 알림톡을 모두 발송한 경우 **나중에 실제 발송이 완료된 시간을 기준으로 전송일시가 표시됩니다.**
  
---  
  
### 2. 전송 이력 조회  
  
전송 이력 탭에서는 다음 정보를 확인할 수 있습니다.  
  
| 항목       | 설명                             |
| -------- | ------------------------------ |
| 유형       | 전송 유형 (개최안내, 안내, 리마인드, 결과안내 등) |
| 전송일시     | 이메일 또는 알림톡이 실제로 발송된 시스템 시간     |
| 조합원명     | 전송 대상 조합원                      |
| 구분       | 개인 / 법인                        |
| 수신자      | 조합원 본인 또는 담당자                  |
| 이메일      | 전송한 이메일과 전송 결과                 |
| 알림톡(SMS) | 전송한 휴대폰 번호와 전송 결과              |
| 전송자      | 전송 버튼을 누른 사용자                  |
  
- 기본 정렬: **전송일시 기준 내림차순**  
- 이메일 및 알림톡 영역에서는 **성공/실패 상태 아이콘**이 함께 표시됩니다.  
  
---  
  
### 3. 전송 상태 표시  
  
각 전송 채널별로 **성공 또는 실패 여부**가 표시됩니다.  
  
- **성공**  
- 성공 아이콘과 함께 전송된 이메일 또는 연락처 표시  
  
- **실패**  
- 실패 아이콘과 함께 전송 대상 정보 표시  
- 실패 아이콘에 마우스를 올리면 **실패 사유** 확인 가능  
  
가능한 실패 사유 예시는 다음과 같습니다.  
  
- 형식 오류  
- 수신 거부  
- 서버 오류  
- 잘못된 번호
- 수신기기 이상

그외 이슈는 고객센터로 문의주시길 바랍니다. 
  
---  
  
### 4. 실패 시 재전송  
  
전송이 실패한 경우 해당 전송을 **개별적으로 재전송**할 수 있습니다.  
  
1. 실패 아이콘 클릭  
2. 전송 실패 안내 모달 확인  
3. **[재전송] 버튼 클릭**  
  
재전송 결과에 따라 다음 메시지가 표시됩니다.  
  
- 성공: `성공적으로 재전송했습니다.`  
- 실패: `재전송에 실패했습니다.`  
  
---  
  
### 5. 이메일 또는 연락처 변경 시 재전송 기준  
  
재전송은 기본적으로 **기존에 전송했던 이메일 또는 연락처로 재시도**합니다.  
  
다만 다음과 같은 경우에는 **변경된 정보로 재전송**할 수 있습니다.  
  
- 이메일 주소 수정  
- 휴대폰 번호 수정  
  
이 경우 전송 실패 모달에서 변경된 정보를 안내한 후 재전송을 진행합니다.  
  
---  
  
### 6. 전송 유형  
  
전송 이력에는 다음과 같은 유형이 표시됩니다.  
  
| 유형 | 설명 |  
|---|---|  
| 개최안내 | 총회 개최 안내 |  
| 안내 | 추가 전송, 수정 전송 등 일반 안내 |  
| 리마인드 | 총회 응답 요청 리마인드 |  
| 취소안내 | 총회 철회 안내 |  
| 마감일연장 | 응답 마감일 연장 안내 |  
| 결과안내 | 총회 결과 안내 |  
  
---  
  
## 자주 묻는 질문  
  
> 전송이 실패했는데 어떻게 다시 보낼 수 있나요?  
{: .prompt-tip }  
  
- 전송 이력에서 **실패 아이콘을 클릭한 후 [재전송] 버튼을 누르면 해당 수신자에게 다시 전송할 수 있습니다.**  
  
> 이메일이나 연락처를 수정한 후 재전송하면 어떤 정보로 전송되나요?  
{: .prompt-tip }  
  
- 수신자의 이메일 또는 연락처가 변경된 경우 **수정된 최신 정보로 재전송됩니다.**

> 전송 버튼을 눌렀는데 전송일시가 바로 표시되지 않습니다.
{: .prompt-tip }

- 전송 버튼을 클릭하면 먼저 전송 이력 행이 생성되고 이후 이메일 또는 알림톡이 실제로 발송됩니다.  
- 실제 발송이 완료되기 전까지는 전송일시가 표시되지 않을 수 있으며, 발송 완료 후 화면을 새로고침하면 전송일시를 확인할 수 있습니다.

> 이메일과 알림톡을 모두 발송한 경우 전송일시는 어떤 기준으로 표시되나요?
{: .prompt-tip }

- 이메일과 알림톡을 모두 발송한 경우 **나중에 실제 발송이 완료된 시간을 기준으로 전송일시가 표시됩니다.**
