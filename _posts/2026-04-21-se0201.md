비즈니스 분야·직위·직책 등 운용사에서 실제 사용하는 명칭과 분류 체계를 VCworks 전반에 반영하기 위한 사용자 코드 설정 화면입니다.

## 설명

- 진입 경로: `설정 > 사용자 코드 관리`
- 관리 가능한 사용자 코드 8종: 비즈니스 분야, 기술 분류, 테마 분류, 가치평가 기준, 가치평가 주체, 직위, 직책, 발굴 경위
- 별도의 사전 준비사항은 없습니다.

#### 사용자 코드 등록/수정

1. 좌측 드롭다운에서 등록할 코드 유형을 선택해주세요.
2. **[+ 신규 등록]** 버튼을 클릭해 항목을 추가하거나, 기존 행을 선택해 수정할 수 있습니다.
3. 이름·사용 여부·상위 항목(해당 시)을 입력한 후 **[저장]** 버튼을 눌러주세요.

    ![저장](assets/img/se0201_02.png)

#### 코드 유형별 용도

| 코드 | 용도 |
|------|------|
| **비즈니스 분야** | 회사별 검토 이력·투자 심의·포트폴리오·BI에서 기업을 비즈니스 분야별로 분류 |
| **기술 분류** | 동일 화면에서 기업의 보유 기술별 분류 |
| **테마 분류** | 동일 화면에서 기업의 테마별 통계 분류 |
| **가치평가 기준** | [가치평가 등록(pm0301·pm0302)](/posts/pm0301/)에서 평가 기준 선택 |
| **가치평가 주체** | 가치평가 등록 시 평가 주체(내부·외부 기관 등) 선택 |
| **직위** | [임직원 등록(hr0001)](/posts/hr0001/) 시 사원·대리·과장 등 직위 지정 |
| **직책** | 임직원 등록 시 팀장·팀원 등 직책 지정 |
| **발굴 경위** | 투자 심의·포트폴리오 정보 관리 등에서 기업 발굴 경로 분류 |

#### 계층 구조

비즈니스 분야처럼 상위-하위가 있는 코드는 **최대 1단계까지** 하위 항목을 설정할 수 있습니다.

![비즈니스 분야 계층 예시](assets/img/se0201_03.png)

## 자주 묻는 질문

> 이미 사용 중인 사용자 코드를 `미사용`으로 변경하면 기존 데이터는 어떻게 되나요?
{: .prompt-tip }
- 기존에 저장된 데이터는 영향을 받지 않고 그대로 유지됩니다.
- 다만 이후 해당 데이터를 수정하게 될 경우, 시스템 정책에 따라 **현재 사용 중인 코드 목록에서만** 수정 값을 선택할 수 있습니다. 미사용으로 전환된 과거 값은 재선택 불가.

> 사용자 코드 관리 메뉴가 `설정`에서 보이지 않습니다.
{: .prompt-tip }
- 로그인한 ID에 `관리자` 역할이 부여된 경우에만 접근 가능합니다. [역할/권한 관리(se0003)](/posts/se0003/)에서 역할 설정을 확인해주세요.

> 하위 항목을 2단계 이상으로 깊게 구성하고 싶습니다.
{: .prompt-tip }
- 현재는 1단계 하위 항목만 지원합니다. 2단계 이상 필요 시 [we@dkdk.kr](mailto:we@dkdk.kr)로 요건과 함께 문의 부탁드립니다.

> 투자 심의·포트폴리오에서 사용할 분류 체계를 바꾸고 싶은데, 등록하면 바로 반영되나요?
{: .prompt-info }
- 저장 즉시 해당 코드를 선택하는 모든 화면(회사별 검토 이력·투자 심의·포트폴리오 정보·BI 등)에 반영됩니다.
- 단 기존 저장 데이터의 분류 값은 영향 없이 유지되며, 새로 입력·수정하는 시점부터 변경된 코드가 선택지로 나타납니다.

버그 및 문의 사항은 다음 이메일로 보내주세요: **[we@dkdk.kr](mailto:we@dkdk.kr)**
